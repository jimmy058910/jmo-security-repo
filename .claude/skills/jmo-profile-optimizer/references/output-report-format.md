# Output Report Format Reference

Template and example for the `OPTIMIZATION_REPORT.md` generated by Phase 7.

---

## Report Template

```markdown
# Profile Optimization Report

**Profile:** balanced
**Scan Date:** 2025-10-21 14:30:00
**Total Duration:** 1215.3s (20.3 min)
**Target Duration:** 900s (15 min) - **35% slower than target**

---

## Baseline Comparison

| Metric | Current | Baseline (2025-09-15) | Change |
|--------|---------|----------------------|--------|
| Total Duration | 1215s | 900s | +35% |
| Trivy Avg Duration | 180s | 145s | +24% |
| Trivy Timeout Rate | 13% | 2% | +11% |
| Trufflehog Avg Duration | 45s | 52s | -13% |

**Status:** **PERFORMANCE REGRESSION** (Trivy timeout rate increased significantly)

---

## Bottlenecks

| Tool | Duration | % of Total | Avg | Timeouts |
|------|----------|------------|-----|----------|
| trivy | 4151.5s | 68% | 180.5s | 3/23 (13%) |
| semgrep | 450.2s | 37% | 37.5s | 0/12 (0%) |

---

## Recommendations

### Immediate Actions (P1)

**1. Increase Trivy Timeout**

    per_tool:
      trivy:
        timeout: 900  # Increased from 600s

**Impact:** Reduce timeout rate from 13% to <5%

**2. Reduce Thread Count**

    profiles:
      balanced:
        threads: 2  # Reduced from 4

**Impact:** Reduce overall scan time by 10-15% (less contention)

### Short-Term Optimizations (P2)

**3. Optimize Trivy Configuration**

- Add `--skip-dirs vendor,node_modules` to exclude dependencies
- Use `--scanners vuln,secret` (skip misconfig for faster scans)
- Enable Trivy cache: `--cache-dir ~/.cache/trivy`

**Expected Impact:** Reduce Trivy time by 20-30%

**4. Optimize Semgrep Configuration**

- Add `--exclude` patterns for test files
- Use `--metrics=off` to disable telemetry

**Expected Impact:** Reduce Semgrep time by 10-15%

### Long-Term Strategy (P3)

**5. Profile Restructuring**

- Move Trivy to 'deep' profile only
- Create 'quick' profile: trufflehog + semgrep (5-8 min)
- Use 'balanced' for pre-merge, 'deep' for nightly scans

**Expected Impact:** CI pipeline: 20min -> <10min (50% faster)

---

---

## Next Steps

1. **Apply configuration changes** (5 min)
2. **Run test scan** to verify improvements (20 min)

4. **Store updated baseline** in memory (automatic)

**Memory Usage:** Baseline loaded from 2025-09-15, updated baseline will be stored after applying changes

**Generated by:** jmo-profile-optimizer v2.1.0
```

## Report Sections Explained

| Section | Purpose |
|---------|---------|
| Header | Profile name, date, duration, target comparison |
| Baseline Comparison | Table comparing current vs historical metrics |
| Bottlenecks | Tools consuming >30% of total scan time |
| Recommendations | Prioritized actions (P1 immediate, P2 short-term, P3 long-term) |
| Updated Configuration | Ready-to-paste jmo.yml changes |
| Next Steps | Checklist for applying and verifying changes |
